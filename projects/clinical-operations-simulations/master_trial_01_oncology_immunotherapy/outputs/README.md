# outputs/README.md

# Outputs

This folder contains **CRA-generated work products** produced during simulation activities.

Outputs represent documentation created as part of monitoring and operational oversight.

---

## Typical Output Types

Examples include:

* Monitoring Visit Reports
* Follow-Up Letters
* Risk Assessments
* Action tracking summaries
* Scenario-level summaries

---

## Purpose

Outputs demonstrate:

* monitoring decisions
* documentation practices
* communication workflows
* evolving risk interpretation over time

These files reflect the execution layer of the simulation.

---

## Working Guidance

* Outputs are generated during scenarios.
* Templates used to create outputs are stored separately in the program-level `templates/` folder.
* Files in this directory should represent completed or in-progress work products.

---

## Organizational Note

Subfolders may be added over time as simulation complexity grows, for example:

```
monitoring_reports/
follow_up_letters/
risk_assessments/
```
